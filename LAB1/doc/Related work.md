

Topic : Reasoning capability in SLMs

Possible Research question :  “How well can small language models learn reasoning via CoT knowledge distillation from a larger teacher, under realistic resource constraints?”

Paper 1 : wei (2023) : Chain-of-Thought Prompting Elicits Reasoning in Large Language Models 

key findings:

* COT performs better on complicated problems ( For example Math) than simpler problems.  
* models of small scale produce fluent but illogical cots: lower performance than standard prompting.  
* scaling improves cot reasoning ability  
* Cot generated after the answer performs same as the baseline: Cot is useful for reasons beyond activating knowledge 

Limitations: 

* few shots setting cost and this can be solved by either synthetic data generation or zero shot generalization  
* There is no guarantee of correct reasoning paths which can lead to correct and incorrect answers

Paper 2 :(Magister 2023): Teaching Small Language Models to Reason 

\=\>Reasoning capability of SLMs doesn’t improve with Cot prompting most producing illogical COT and cot prompting even reduces the accuracy of models with less than 10 billions params.

\=\> Explore the transfer of reasoning capabilities to SLMs via knowledge distillation, also investigating model and dataset size trade off.

\=\>perform knowledge distillation by fine tune a student model on cot outputs generated by a larger teacher

Related work mentioned in this paper: 

* wei et al (2022) : COT prompting \=\> prefix an input with 2-8 exemplars of cot cot reasoning encourage the LM to do the same , Reach state of the 	art accuracy on GSM8K.  
* Wang et al (2022) : Task accuracy can further be improved by using self consistency in cot prompting: Samples CoT reasoning paths from model decoder and returns most consistent path by taking the majority vote.  
* chung et al(2022) : explore finetuning flow based version of palm on manually generated COT data.

Method: 

2 step pipeline COT distillation:

1. annotate existing supervised dataset with COT reasoning generated by a teacher model : we use palm 540B and GPT3 175B because COt proven to perform better on large scale LLms / perform 8 shot prompting to genrate COt but key change from previous work: provide the LLM with the target answer / Keep only correct Cot based on answer  
2. Fine tune student model via teacher forcing : Input : question / output(target) : COT \+ answer

Experimental set up : 

Benchmarks : datasets : Arithmetic/ Commonsense / symbolic

Standard prompting : giving question/answer pairs to Lm before outputting the prediction→ the model gives the answer directly

COT prompting: few shot prompting(question/answer pairs) \+ COT from an associated answer

benchmarks and metrics : 

GSM8K : arithmetic reasoning : 10% of data for validation

evaluate task accuracy based on : Target answer \= final answer in COT / use external calculator to handle arithmetic mistakes made by the model and then check the target

Startegy QA : Commonsense reasoning : 80% training data 10% validation % test

we compute task accuracy : accuracy slightly improved from 68% to 71% 

\=\> for strategy A ; the model finetuned on golden COt( provided within the dataset) performs Best Better than Teacher generated COT.

key findings: 

* The proposed method improves task performance across arithmetic commonsense and symbolic datasets : Accuracy increased from 8% to 21% and 18% on GSM8K dataset when T5 XXL finetuned on Palm 540B and GPT3 175B generated COts  
* tradeoff : performance gain vs model size vs dataset size : 20% data achieve 6% more efficient accuracy.

Paper 3 : Wang et al (2023) SELF-CONSISTENCY IMPROVES CHAIN OF THOUGHT REASONING IN LANGUAGE MODELS 

\=\> New decoding strategy : Self consistency to replace ,naive greedy decoding : It samples a diverse set of reasoning paths and then selects the most consistent answer by marginalizing out of the sampled reasoning paths : GSM8K achieved \+17.9% accuracy when using self consistency

Proposed method :” Sample and marginalize” Decoding technique

1. Sample from LM decoder to generate diverse reasoning set of reasoning paths  
2. determine the optimal answer by marginalizing out of the sampled reasoning paths

Paper 4 : ThinkSLM (2025) : THINKSLM: Towards Reasoning in Small Language Models 

Key Findings:

* Reasoning ability in SLMs depends strongly on training recipe+ data quality not only on parameter count   
* Instruction tuned \+ high quality distillation( Strong teacher forcing distillation) are crucial.  
* Compression: Quantization is safe 8bit/4bit  
* Pruning is harmful  
* Prompting (few shot/Zero shot) helps than people think 

**\=\>** Student: Qwen2.5‑3B‑Instruct THINKSLM shows Qwen2.5‑3B (instruction‑tuned/distilled) achieving about 84–85% on GSM8K, high 80s/90s on ARC‑Easy, and low‑80s on commonsense, with strong overall reasoning scores 

* 8‑bit quantization of Qwen2.5‑3B barely changes performance; 4‑bit GPTQ leads to small drops but still excellent scores.

Qwen2.5‑32B‑Instruct (teacher)

* In THINKSLM, Qwen2.5‑32B scores \~95–96% on GSM8K, high 90s on ARC, mid‑90s on CommonsenseQA, and \~87% on sorting, with overall average \>92%.  
  * 4‑bit or 8‑bit quantization preserves its reasoning quality extremely well, which is perfect if you have only \~24 GB VRAM.  
  * Using the same family (Qwen2.5) for teacher and student simplifies tokenization and style matching for CoT.

What is Quantization  : storing some of the model weights in lower precision format such as INT8 and INT8 instead of FP32 or FP16. \=\> reduces memory foot print and storage needs. different types : Post training/ Activation aware.

Paper 5 : Ho et al (2023) **;** Large Language Models Are Reasoning Teachers 

Method : 

1. Fine tune COt generate reasoning samples from very large LLms to fine tune SLms  
2. Generate multiple distinct rationales using teacher LLm to enrich the fine tuning data with diverse reasoning 

\=\> prompting a very large language model to solve complex tasks via zero shot chain of thought reasoning and then fine tune a small student model on this: 

Prompting the teacher \=\> provide COT demonstrations as exemplars in prompting or provide a simple Instruction “Let’s think step by step” 

In this paper we are using zero-shot prompting

Key findings : 

1. Diverse reasoning: enables remarkable gains in performance at the minor cost of additional teacher inference  
2. Filter CoT : 

Final answer \= ground truth (Huang et al (2022)) 

Rationale filtering : Manually annotate the correctness of rationales from teacher model and evaluate the student model performance when fine tuned on correctly reasoned samples : we found that 28% of correct samples have incorrect rationales 

\=\> fine tuning on golden samples( Human reviewed° outperform answer based filtering by 5-8% but there is a trade off between the quality and the quantity of reasoning samples.

\=\> in arithmetic tasks a larger portion of errors arises from calculation  : for instance MathArith and SVAMP show many semantic errors

\=\> on GSM8K all te methods tend to struggle

\=\> common maximum sequence length often lead to incomplete answers \-\>\> incorrect rationels
